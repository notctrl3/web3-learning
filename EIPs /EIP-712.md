## EIP-712

### Motivation                            
 提高链下签名使用，减少gas费和交易次数。结构化签名数据方便展示让用户理解
### Specification
 `Type Structured Data`: 将用户签名数据结构化，方便展示给用户 
 ```
 struct Permit {
    address owner;
    address spender;
    uint256 value;
    uint256 nonce;
    uint256 deadline;
}
```

 `Domain Separator`: 明确签名生效上下文,包括合约地址、链 ID、版本等
 ```
  {
  "name": "MyToken",
  "version": "1",
  "chainId": 1,
  "verifyingContract": "0xTokenAddress"
}
```
### Security Considerations
#### Replay attacks
  EIP712没有对重放攻击的考虑，一般需要合约设置nonce，deadline来避免
#### Frontrunning Attacks
  用户把签名发送到链上执行，攻击者看到了这个交易 → 抢先提交相同签名。通过nonce，白名单，验证msg.sender来避免
