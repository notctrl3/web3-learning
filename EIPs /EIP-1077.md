## [EIP-1077](https://eips.ethereum.org/EIPS/eip-1077):Gas relay for contract calls

### Motivation                            
 - 提出了一个允许通过签名执行任意合约调用的通用方法
### Specification
#### executeGasRelay 
relayer 接收用户签名的“执行意图”，验证签名 → 执行真实的合约调用
``` solidity
function executeGasRelay(
  bytes calldata _execData, // 用户想执行的真实 calldata
  uint256 _gasPrice,        // 支付给 relay 的价格
  uint256 _gasLimit,        // 用户预估 gas
  address _gasToken,        // 用什么 token 付 gas
  address _gasRelayer,      // 费用接收者——谁帮发的交易
  bytes calldata _signature // 用户签名
 ) external;
```

#### executeGasRelay
把“用户将要签名的所有信息”按统一、不可歧义的规则规范化，生成一段确定的 message
``` solidity
function executeGasRelayMsg(
 uint256 _nonce,
 bytes memory _execData,   
 uint256 _gasPrice,
 uint256 _gasLimit,
 address _gasToken,
 address _gasRelayer)
public pure returns (bytes memory);`
```
#### executeGasRelayERC191Msg
用 EIP-191 格式生成用户签名消息
``` solidity
function executeGasRelayERC191Msg(
  uint256 _nonce,
  bytes memory _execData,
  uint256 _gasPrice,
  uint256 _gasLimit,
  address _gasToken,
  address _gasRelayer) public view returns (bytes memory);
```
#### lastNonce
返回当前可用 nonce
``` solidity
function lastNonce() public returns (uint nonce);
```
签名消息内容包括：目标合约/调用数据 (_execData)、gasPrice、gasLimit、gasToken、gasRelayer、nonce 等。
合约在收到 relay 调用时，会验证签名 (符合 EIP-191, 或者 EIP-1271 的 contract 签名验证) 再执行
