## [EIP-1967](https://eips.ethereum.org/EIPS/eip-1967): Proxy Storage Slots

### Motivation                            
 统一规定了proxy几个关键存储槽（slot）的位置
### Specification
 | 功能                 | Slot（bytes32）                                                     | 说明                         |
| ------------------ | ----------------------------------------------------------------- | -------------------------- |
| **Implementation** | `0x360894a13ba1a3210667c828492db98dca3e2076cc3735a920a3ca505d382bbc` (obtained as `bytes32(uint256(keccak256('eip1967.proxy.implementation')) - 1))`. | Proxy delegatecall 的逻辑合约地址 |
| **Admin**          | `0xb53127684a568b3173ae13b9f8a6016e243e63b6e8ee1178d6a717850b5d6103` (obtained as `bytes32(uint256(keccak256('eip1967.proxy.admin')) - 1))`.          | Proxy 的管理员地址，负责升级          |

### Application
#### Transparent Proxy
管理员对代理发的调用不会被转发到实现合约，非管理员的调用才会 fallback 进入实现合约。[USDC](https://github.com/circlefin/stablecoin-evm/blob/master/contracts/upgradeability/AdminUpgradeabilityProxy.sol)
#### UUPS
升级逻辑被迁移到逻辑合约中
